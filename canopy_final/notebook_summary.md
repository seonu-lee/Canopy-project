## 2. `02_preprocessing.ipynb`

### 1. 파일명 및 목적
축3 분석에 필요한 변수(운영시간, 자치구, EV 등록대수)를 축2 전처리 데이터에 추가하고, 동작대교(위) 중복 1건을 제거해 741개의 최종 전처리 파일 생성.

### 2. 주요 데이터셋
| 데이터 | 출처 | 주요 컬럼 |
|--------|------|---------|
| 축2 전처리 데이터 | `parking_preprocessed.csv` | pklt_nm, tpkct, lat, lot, addr |
| OA-13122 원본 | `서울시 공영주차장 안내 정보(원본).csv` (3,696행) | 평일/주말 운영 시작·종료시각(HHMM) |
| EV 등록현황 | `서울시 자치구별 연료별 차종별 용도별 등록현황(26년3월).csv` | 시군구명, 연료명, 계 |

### 3. 핵심 분석 단계
1. **중복 제거 (Step 0)**: 동작대교(위) 공영주차장 1면짜리 행 제거 → 742 → **741개**
2. **운영시간 계산 (Step 1)**: HHMM 정수 → 시간 단위 변환, 2400(자정) 예외처리
   ```python
   def calc_hours(start, end):
       s = int(str(int(start)).zfill(4)[:2])
       e = 24 if int(end) == 2400 else int(str(int(end)).zfill(4)[:2])
       return e - s if (e - s) > 0 else 24
   ```
3. **주차장명 정규화 조인 (Step 2)**: (시)·(구)·괄호 제거, 공백 제거 후 키 매칭
   ```python
   def normalize_name(name):
       name = re.sub(r'\(시\)|\(구\)|舊', '', str(name))
       return re.sub(r'\s+', '', re.sub(r'\([^)]*\)', '', name)).strip()
   ```
4. **가중평균 운영시간 산출 (Step 3)**: `(평일×5 + 주말×2) / 7`
5. **자치구 추출 (Step 4)**: `addr.str.extract(r'(\S+구)')`
6. **EV 등록대수 조인 (Step 5)**: 자치구별 전기차 등록대수 left join

### 4. 주요 결과 및 결론
| 항목 | 결과 |
|------|------|
| 최종 행 수 | 741개 (전체 결측 없음) |
| 가중평균 운영시간 평균 | 17.9시간 (중앙값 24시간, 24시간 운영 비율 높음) |
| 최솟값 운영시간 | 7.14시간 (구룡산제1·2호: 평일 10시간, 주말 미운영) |
| 출력 파일 | `parking_axis3_preprocessed.csv` (10개 컬럼) |

### 5. 개선 아이디어
- OA-13122 원본에 동일 주차장명 여러 행이 존재(3,696행 → 1,694 고유명). 현재 중앙값으로 처리하나, 최신 동기화 시각 기준으로 최신 행 선택하는 방식이 더 정확할 수 있음
- EV 등록대수를 자치구 단위가 아닌 500m 반경 집계로 교체하면 공간 정밀도 향상

## 3. `03_analysis_axis3.ipynb`

### 1. 파일명 및 목적
서울시 공영주차장 태양광 캐노피 설치 시 **자가소비 적합성(축3)**을 정량 평가하는 핵심 점수 산출 노트북. 자체소비 지표(운영시간·노외여부)와 EV충전 지표(50m 반경 충전기 수·200m 반경 충전기 수·EV 등록대수)를 가중 결합해 0~1 점수 생성.

### 2. 주요 데이터셋
| 데이터 | 파일명 | 행수 |
|--------|--------|------|
| 전처리 공영주차장 | `parking_axis3_preprocessed.csv` | 741 |
| EV 충전소 (공공데이터) | `서울_EV충전소.csv` | 74,024 |
| 태양광 설치 주차장 | `solar_parking_list.csv` | 74 |
| EV 등록현황 | OA-15640 (2026.3월) | 25개 자치구 |

### 3. 핵심 분석 단계
1. **EV 충전소 좌표 정제**: 74,024개 전처리 후 하버사인 공식으로 50m/200m 반경 집계
   ```python
   def haversine_vectorized(lat1, lon1, lat2, lon2):
       R = 6371000
       # ... (벡터화 구현)
       return R * 2 * np.arctan2(np.sqrt(a), np.sqrt(1-a))
   ```
2. **지표 정규화**: Min-Max Normalization (0~1)
3. **점수 산출**:
   - 자체소비 점수 = `0.6 × 운영시간_norm + 0.4 × 노외여부`
   - EV충전 점수 = `0.4 × ev_50m_norm + 0.3 × ev_200m_norm + 0.3 × ev등록_norm`
   - **축3 최종** = `0.7 × 자체소비_점수 + 0.3 × EV충전_점수`
4. **자치구별·유형별 분포 분석**: 박스플롯, 집계 통계

### 4. 주요 결과 및 결론
| 통계량 | 값 |
|--------|-----|
| 평균 | 0.529 |
| 중앙값 | 0.703 |
| 표준편차 | 0.272 |
| 노외 중앙값 | 0.80 |
| 노상 중앙값 | 0.15 |

- **상위 자치구**: 성동구(0.728), 서초구(0.725), 성북구(0.716)
- **하위 자치구**: 서대문구(0.360), 영등포구(0.327), 노원구(0.325)  
- **Top 3 주차장**: 영등포동제2공영(1.000), 잠실역(0.932), 수서역북(0.918)
- 노상/노외 점수 간 약 0.52점 차이 → 노외가 압도적 우위
- 출력 파일: `parking_axis3_scored.csv` (741행, 14컬럼)

### 5. 개선 아이디어
- EV 충전기 50m 반경은 서울 도심 밀도에서 과잉추정 위험 → **POI 필터링**(충전소 유형·운영 상태) 결합 검토
- 노외여부를 이진 변수로만 사용 중. 노외 주차장 내에서도 개방형/폐쇄형 구분을 추가하면 변별력 향상
- EV 등록대수를 자치구 단위에서 동 단위로 세분화하면 공간 해상도 개선 가능

## 5. `05_validation.ipynb`

### 1. 파일명 및 목적
축3 점수 산출에 사용한 **50m 프록시(공공데이터 기반 EV 충전기 반경 집계)**의 정확도를 실측 데이터(웹 크롤링 + 서울시 리서치 + 시설공단 리스트)와 비교·검증하는 노트북.

### 2. 주요 데이터셋
| 데이터 | 설명 | 행수 |
|--------|------|------|
| `parking_axis3_scored.csv` | 모델 추정치 | 741 |
| `seoul_parking_simple_list.csv` | 일렉베리 크롤링 (공영주차장 필터) | 144 |
| `seoul_parking_geocoded.csv` | 크롤링 데이터 Vworld 지오코딩 결과 | 144 |
| 서울시 리서치 리스트 | 수동 조사 9개소 | 9 |
| 서울시설공단 리스트 | 과거 매칭 리스트 25개소 | 25 |

### 3. 핵심 분석 단계
1. **일렉베리 크롤링**: Selenium으로 15페이지(144건) 수집
   ```python
   url = "https://www.elecvery.com/ko/map?placeTypeList=PUB_PARKING"
   # 페이지 버튼 자동 클릭 + 리스트 파싱
   ```
2. **텍스트 매칭 검증**: 주차장명 공백 제거 후 left join → Precision 0.05
3. **좌표 기반 검증 (50m)**: Vworld API 지오코딩 + 하버사인 거리 매칭 → Precision 0.03
4. **좌표 기반 검증 (20m)**: 임계값 축소 → Precision 0.01
5. **3종 데이터 통합**: 일렉베리 50m + 리서치 9개 + 시설공단 25개 → 통합 실측값 30건 구축
6. **KML 생성**: Google Earth 4색 핀 시각화 (Case A/B/C/D)

### 4. 주요 결과 및 결론
| 검증 방법 | TP | FP | FN | Precision |
|---------|----|----|----|----|
| 텍스트 매칭 | 6 | 110 | 28 | 0.05 |
| 좌표 50m | 3 | 113 | 0 | 0.03 |
| 좌표 20m | 1 | 115 | 0 | 0.01 |
| **통합 실측 (30개소)** | — | — | — | — |

- **낮은 정합성의 원인**: ① 일렉베리 커버리지 극히 일부(144건/전체 대비 소수), ② 50m 반경 내 도심 빌딩·아파트 충전기 혼입(과잉 추정), ③ 대형 부지 주차장의 좌표 오차
- 결론: 일렉베리 단일 검증은 폐기. 통합 실측(30개소)으로 공간적 오차 원인을 Google Earth KML로 시각화해 구조적 한계 증명

### 5. 개선 아이디어
- 환경부 전기차 충전소 API(전수 데이터)와 좌표 매칭 병행 시 검증 표본 수 대폭 확대 가능
- 50m 반경 대신 **폴리곤 기반 필지 경계** 내 충전소 집계로 과잉 추정 해결 가능
- Vworld 지오코딩 실패 케이스(주소 파싱 오류)에 대한 수동 보정 로직 추가 권장

## 6. `서울시_태양광주차장_입지선정지표_hybrid.ipynb`

### 1. 파일명 및 목적
축1(태양광 발전량), 축2(유휴공간 활용), 축3(자가소비 적합성), 축4(공공편익)의 4개 축 점수를 **AHP(주관) × 엔트로피(객관) 하이브리드 가중치**로 결합해 741개 공영주차장의 **최종 종합 순위**를 산출하고 시각화하는 최종 통합 노트북.

### 2. 주요 데이터셋
| 축 | 입력 파일 | 점수 컬럼 |
|----|-----------|---------|
| 축1 | `축1_results.csv` | `final_score` |
| 축2 | `축2_results.csv` | `total_score` |
| 축3 | `축3_results.csv` | `축3_자가소비적합성` |
| 축4 | `축4_results.csv` | `공공편익점수` |

### 3. 핵심 분석 단계
1. **4축 데이터 결합**: `pklt_nm` 기준 inner join → 741개
2. **AHP 가중치 산출**: 쌍대비교 행렬 → 정규화
   ```python
   matrix = np.array([
       [1,   3,   5,   7],  # 축1: 태양광 (가장 중요)
       [1/3, 1,   2,   4],  # 축2: 유휴공간
       [1/5, 1/2, 1,   2],  # 축3: 전기 활용
       [1/7, 1/4, 1/2, 1],  # 축4: 복지 활용 (후순위)
   ])
   ahp_weights = (matrix / col_sums).mean(axis=1)
   ```
3. **엔트로피 가중치 산출**: 데이터 분산 기반 객관 가중치
   ```python
   P = X_safe / X_safe.sum(axis=0)
   E = -k * np.sum(P * np.log(P), axis=0)   # k = 1/ln(n)
   entropy_weights = (1 - E) / (1 - E).sum()
   ```
4. **하이브리드 가중치**: `AHP × 엔트로피 → 재정규화`
5. **최종 점수 산출**: `Σ(축k_score × hybrid_weight_k)`
6. **시각화**: 코로플레스 지도 + 상위 10개 핀 마킹, 상위 15개 요약표, 4축 히트맵(BuPu/YlOrRd), 게이지 차트
